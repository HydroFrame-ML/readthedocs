hydrodata.data_catalog.validate_hydrodata_catalog module

Functions to validate the hydrodata_catalog.yaml file.

hydrodata.data_catalog.validate_hydrodata_catalog.ALLOWED_ATTRIBUTES = ['description', 'dataset_var', 'title', 'start_date', 'end_date', 'datapath']

Global constant. Attributes without validation tables that are allowed in a data_catalog entry.

hydrodata.data_catalog.validate_hydrodata_catalog.validate_data_catalog_yaml(fix_option=None)

Validate the hydrodata_catalog.yaml file.

Parameters:

fix_option – If None or “fix” then fix the yaml file with missing attributes.

This prints an error and a fails with a non-zero return code if the yaml file is inconsistent. The file is inconsistent if there exists a column in the data_catalog_entry tables that is not a dimension table and is not in the ALLOWED_ATTRIBUTS list. It is also inconsistent if any column in a table with the same name as a dimension table has a value that is not in that dimension.

Returns:

Exit with an non-zero status code if a validation error is found or a fix to the yaml file was required.

A data catalog is fixed by finding attributes in the data_catalog_entry table that have a name with a dimension table and the dimension row of the data_catalog_entry table value contains other attributes that are also dimension table. In this case the other dimension attributes from the dimension table are assigned to the same dimension attribute in the data_catalog_entry table.
